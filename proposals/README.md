# AreaFlow Proposals

本目录保存尚未成为当前产品事实的重大设计。Proposal 用于讨论目标、风险、边界和开闸条件；只有实现、测试、验证和长期文档全部完成后，对应能力才能从 proposal 迁入 `docs/**`。

状态约定：

- `Proposed`：正在设计，未批准实现。
- `Accepted`：设计已批准，但不代表功能可用。
- `Implemented`：实现和验证完成，当前行为已经回写长期文档。
- `Deferred`：明确推迟。
- `Superseded`：被其他 proposal 或 ADR 替代。

当前 proposals 均属于尚未开放或只完成边界预留的能力。
