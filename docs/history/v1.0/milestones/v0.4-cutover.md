# v0.4 Workflow Ownership Cutover

阶段合同见 [`../architecture/v0.4-workflow-ownership-cutover-contract.md`](../contracts/v0.4-workflow-ownership-cutover-contract.md)。
本 milestone 只切 AreaFlow DB 内的新 workflow version authoring ownership，不切 execution、不转发
`./task-loop run`、不修改 AreaMatrix shim 文件，也不写 `workflow/versions/**`。

## Goal

新 workflow version 的源事实切换到 AreaFlow。

## Includes

- AreaMatrix compatibility command design。
- Compatibility contract API/CLI。
- New versions owned by AreaFlow。
- AreaMatrix summary export。
- Legacy workflow read-only mode。
- Approval gate。
- Live mapping gate。
- Profile binding drift gate。
- Cutover readiness bundle/API/CLI。
- Cutover readiness gate。
- Protected `project.cutover.apply` for DB-only authoring cutover。

## Excludes

- task-loop replacement。
- 自动代码修改。
- promotion apply。
- execution package 写入。
- AreaMatrix shim file edits without separate explicit authorization。
- execution cutover or `./task-loop run` forwarding。

## Success

- 新 workflow version 的源事实可切到 AreaFlow。
- AreaMatrix 保留 `workflow/README.md` 与 `.areaflow/status.json` 粗略入口。
- legacy workflow 进入 read-only / immutable 口径。
- compatibility commands 能转发或降级展示 AreaFlow 状态。
- `areaflow project compatibility <project>` 能输出 shim contract。
- cutover 前检查 import coverage、hash drift、stage coverage、doctor 等价、权限 allowlist 和 audit_events。
- `profile_binding_drift` 能证明 authored workflow version 冻结的 profile hash 与当前 profile registry 是否一致。
- `approval_gate` 能证明用户显式 approval 已经存在且 linked transition preview 为 ready。
- `live_mapping_gate` 能证明 approval、transition preview 和 promotion preview gate 串联一致。
- `areaflow project cutover-readiness <project> --version <label>` 能输出切换前 bundle。
- `GET /api/v1/projects/{project_key}/cutover-readiness?version=<label>` 能返回同一 bundle。
- `cutover_readiness_gate` 能把 verification、compatibility、approval 和 live mapping 证据写入 gate result。
- rollback plan 明确，rollback 追加事件而不是删除历史。
- `project cutover-apply` 只更新 AreaFlow workflow version lifecycle/status summary，返回
  `project_write_attempted=false` 和 `execution_write_attempted=false`。
- 非 `authoring_cutover` mode 必须 blocked。

## v0.4a Boundary

v0.4a 先落地 cutover 前的 profile binding 和 approval gate：

- `areaflow workflow gate run <project> <label> profile_binding_drift`
- `areaflow workflow gate run <project> <label> approval_gate`
- `areaflow workflow gate run <project> <label> live_mapping_gate`

`profile_binding_drift` 只检查 authored workflow version 冻结的 `profile_binding` 是否仍匹配当前
profile registry。hash/version 不一致时返回 `warn`，缺少绑定或无法加载当前 profile 时返回
`blocked`。它不修改 workflow version，不自动迁移 profile，也不改变 gate 规则。

`approval_gate` 只检查最新 approval record 是否为 `approved`，并确认它关联的 transition preview
为 `ready`。

`live_mapping_gate` 只检查以下链路是否可证明：

```text
promotion_preview gate pass
-> transition preview ready
-> approval record approved
```

v0.4a 仍不执行 promotion apply，不写 AreaMatrix execution，不创建 runner package。
即使 `live_mapping_gate` 为 `pass`，也只是证明后续 cutover/promote 的前置条件，不代表已经执行。

## v0.4b Boundary

v0.4b 落地 compatibility command contract：

- `areaflow project compatibility <project>` 输出 AreaMatrix shim 应如何处理每个保留命令。
- REST API 提供 `GET /api/v1/projects/{project_key}/compatibility`。
- contract command mode 包含：
  - `forward`：优先转发到 AreaFlow CLI/API。
  - `fallback_status`：AreaFlow/native 条件不足时读取 `.areaflow/status.json` 展示降级状态。
  - `blocked`：v0.4 明确禁止执行。
- `.areaflow/status.json` 增加轻量 `compatibility` summary，供离线 shim 降级展示。
- `./task-loop run`、promotion apply、write execution、git checkpoint 继续禁止自动转发。
- AreaMatrix 侧最小 shim 落地计划见
  [`../migration/areamatrix-compatibility-shim-plan.md`](../migrations/areamatrix-compatibility-shim-plan.md)。

v0.4b 仍不修改 AreaMatrix shim、不写 `workflow/README.md`，只把 AreaFlow 侧 contract 做成可查询、
可测试的源事实。

## v0.4c Boundary

v0.4c 落地 cutover readiness bundle 和 gate：

- `areaflow project cutover-readiness <project> --version <label>` 输出 cutover 前聚合检查。
- REST API 提供 `GET /api/v1/projects/{project_key}/cutover-readiness?version=<label>`。
- `areaflow workflow gate run <project> <label> cutover_readiness_gate` 把检查结果写入 `gate_results`。
- bundle 聚合：
  - v0.2 verification bundle。
  - status mirror。
  - compatibility contract。
  - authored workflow version。
  - latest `approval_gate`。
  - latest `live_mapping_gate`。
  - rollback plan 文档来源。

v0.4c 仍然只读。`cutover_readiness_gate` 即使 pass，也不执行 cutover apply、不写 execution package、
不替代 `./task-loop run`，也不修改 AreaMatrix shim。

## v0.4d Boundary

v0.4d 落地受保护的 authoring cutover apply：

- `areaflow project cutover-apply <project> --version <label>` 只允许 `mode=authoring_cutover`。
- REST API 提供 `POST /api/v1/projects/{project_key}/cutover-apply`。
- apply 前必须已有 `project cutover-readiness` phase gate pass 和 `cutover_readiness_gate` pass。
- apply 后只更新 AreaFlow PostgreSQL 中该 workflow version 的 `lifecycle_status=authoring_cutover` 与
  `status_summary.authoring_cutover`。
- command response、event 和 audit event 必须记录 `project_write_attempted=false`、
  `execution_write_attempted=false` 和 `area_matrix_write_attempted=false`。

v0.4d 仍不写 AreaMatrix `workflow/README.md`、`.areaflow/status.json`、shim 文件或
`workflow/versions/**`，也不替代 `./task-loop run`。
