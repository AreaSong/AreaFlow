# v0.7 Web Dashboard

## Goal

提供多项目状态可视化。
阶段合同见 [`../architecture/v0.7-web-dashboard-contract.md`](../contracts/v0.7-web-dashboard-contract.md)。
本阶段 Web 是 `/api/v1` + SSE 的观察面，不是第二状态源，也不是写操作入口。

## Includes

- React + TypeScript dashboard。
- Vite web app under `web/`, with `/api` proxied to the AreaFlow service.
- Project list backed by `GET /api/v1/projects`。
- Project detail backed by `GET /api/v1/projects/{project_key}`。
- Project summary first screen backed by `GET /api/v1/projects/{project_key}/summary`,
  including stable doctor status, drift status, and stage coverage status.
- Version timeline。
- Stage view。
- Run timeline backed by
  `GET /api/v1/projects/{project_key}/workflow-versions/{version}/runs?limit=20`.
- Artifact browser。
- Version detail artifact/residual filters:
  `GET /api/v1/projects/{project_key}/workflow-versions/{version}/artifacts?limit=20` and
  `GET /api/v1/projects/{project_key}/workflow-versions/{version}/residuals?limit=20`.
- Read-only trace detail panel linking stage, workflow item, artifact, residual, and approval context.
- Read-only run detail API: `GET /api/v1/runs/{run_id}?project_key={project_key}` with tasks,
  attempts, artifact metadata, and global-ID visibility guard.
- Read-only project artifact browser API: `GET /api/v1/projects/{project_key}/artifacts?limit=20`.
- Read-only artifact metadata API: `GET /api/v1/artifacts/{artifact_id}`.
- Blocker/residual view backed by `GET /api/v1/projects/{project_key}/residuals?limit=20`.
- Read-only approval records panel backed by
  `GET /api/v1/projects/{project_key}/workflow-versions/{version}/approvals?limit=20`.
- Read-only worker status panel backed by `GET /api/v1/projects/{project_key}/workers?limit=20`.
- Read-only audit trail panel backed by `GET /api/v1/audit-events?project_key={project_key}&limit=20`.
- SSE event streams: `GET /api/v1/events/stream`, `GET /api/v1/projects/{project_key}/events/stream`,
  and `GET /api/v1/runs/{run_id}/events/stream`, all backed by `events`.

## Excludes

- Web 不维护状态源事实。
- Web 不直接写项目文件。
- Web 不绕过 AreaFlow API。
- Web 初始实现不发送非 GET `/api/v1` 请求。
- Web 不把 SSE event 当作主状态。
- Web 不把 `web/write-action-gate` 的 disabled action 当作许可。

## Success

- Web 首屏只依赖 AreaFlow summary/readiness API。
- Stage、artifact、residual 和 approval 的联动详情只读展示现有 API metadata，不读取 artifact 原文。
- Worker status 只读展示 worker registry / heartbeat metadata，不触发 heartbeat、lease 或 run-once。
- Audit trail 只读展示 action、decision、capability、resource、reason 和 created_at，不触发权限判断、
  worker command 或任何项目写入。
- Timeline 和 run 状态通过 events/SSE 更新，SSE 不维护第二套状态。
- Approval、drain、cancel 等操作走 command API；v0.7 初始 Web 先展示 approval records，不发起
  写操作。写入动作必须等 Command API、risk preview、permission preflight、影响范围和 audit outcome
  稳定后再打开。
- 高风险动作展示 capability、影响路径、approval 和 audit 结果。
- Browser smoke 证明 Web 没有发出非 GET `/api/v1` 请求，run detail 带 `?project_key=` visibility guard。
