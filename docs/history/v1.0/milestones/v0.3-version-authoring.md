# v0.3 New Version Authoring

阶段合同见 [`../architecture/v0.3-version-authoring-contract.md`](../contracts/v0.3-version-authoring-contract.md)。
本 milestone 只建立 AreaFlow-owned authoring state，不打开 authoring cutover、promotion apply、execution
cutover、worker execution 或 managed project write。

## Goal

AreaFlow 能创建新 workflow version 候选并生成 discussion 到 promotion preview 的 artifact。

## Includes

- Version creation。
- Discussion gate authoring。
- Middle-layer/change/plan/draft/queue candidate。
- Promotion preview。
- Approval record model。

## Excludes

- 写 AreaMatrix live execution。
- 执行任务。
- 写被管理项目 `workflow/versions/**`。
- 把 approval record 解释为 execution permission。

## v0.3a Boundary

v0.3a 先落地 DB-only workflow version candidate：

- `areaflow workflow version create <project> <label>` 创建 AreaFlow-owned draft。
- `areaflow workflow version list <project>` 查询项目下 workflow versions。
- `areaflow workflow version show <project> <label>` 查询单个 workflow version。
- REST API 提供 `GET/POST /api/v1/projects/{project_key}/workflow-versions` 和
  `GET /api/v1/projects/{project_key}/workflow-versions/{label}`。
- 创建时写入 `workflow_versions(import_mode='authored')`、`workflow_items(stage='version_init')`、
  `events`、`audit_events` 和 `command_requests`。
- 同一 `idempotency_key` 重复请求返回同一结果；同一 key 携带不同 request hash 必须拒绝。
- AreaMatrix import 只能刷新 `metadata_only` 的 imported index，不能删除 `authored` versions。

v0.3a 不生成 discussion 文件、不生成 prompts、不写被管理项目 `workflow/versions/**`、不做
promotion apply、不调用 AI engine。

## Later v0.3 Slices

- v0.3b：生成 discussion、middle-layer、changes、plans、drafts、queue 和 promotion preview
  的 AreaFlow artifact 与 workflow item。
- v0.3c：把 workflow gate result 提升为可查询、可审计的系统约束。
- v0.3d：把 transition preview 和 approval record 提升为可查询、可审计的系统约束。

## v0.3b Boundary

v0.3b 在 AreaFlow artifact store 中生成 stage skeleton：

- `areaflow workflow version create <project> <label>` 自动创建 stage skeleton。
- `areaflow workflow version stages <project> <label>` 查询 version 下的 workflow items。
- `areaflow workflow version ensure-skeleton <project> <label>` 为已有 authored version 补齐 skeleton。
- REST API 提供 `GET/POST /api/v1/projects/{project_key}/workflow-versions/{label}/stages`。
- stage skeleton 包含 `discussion`、`middle_layer`、`changes`、`plans`、`drafts`、`queue` 和
  `promotion_preview`。
- `draft_copy` 与 `draft_verify` 是不同 workflow item 和不同 artifact type。
- artifact 原文写入 AreaFlow 本地 artifact store，DB 保存 URI、hash、type、size 和 item relation。

v0.3b 仍不写被管理项目 `workflow/versions/**`，也不进入 approval、promotion apply 或 execution。

## v0.3c Boundary

v0.3c 先落地通用 workflow gate result：

- `areaflow workflow gate run <project> <label> <gate_name>` 记录一次 gate 判断。
- `areaflow workflow gate list <project> <label>` 查询 gate result timeline。
- REST API 提供 `GET/POST /api/v1/projects/{project_key}/workflow-versions/{label}/gates`。
- `gate_results` 记录 gate name、scope、status、inputs、warnings、failures、metadata 和 checked time。
- v0.3c 支持 `discussion_gate`、`plan_doctor`、`draft_doctor`、`queue_doctor` 和
  `promotion_preview`。
- 缺少 required workflow item 时 gate 结果为 `blocked`。
- skeleton-only artifact 只能得到 `fail`，不能自动通过，也不能推动下游 item 进入 `ready`。

v0.3c 仍不做 approval、不做 promotion apply、不写被管理项目、不执行任务。

## v0.3d Boundary

v0.3d 落地 transition preview 和 approval record：

- `areaflow workflow transition preview <project> <label>` 记录一次从 `promotion_preview`
  到 `approval` 的只读预演。
- `areaflow workflow transition list <project> <label>` 查询 transition preview timeline。
- `areaflow workflow approval record <project> <label>` 记录一次显式批准或拒绝。
- `areaflow workflow approval list <project> <label>` 查询 approval timeline。
- REST API 提供：
  - `GET/POST /api/v1/projects/{project_key}/workflow-versions/{label}/transition-previews`
  - `GET/POST /api/v1/projects/{project_key}/workflow-versions/{label}/approvals`
- `workflow_transition_previews` 记录 from/to stage、required gate、latest gate result、
  blockers、warnings、metadata 和 created time。
- `approval_records` 记录 approval kind、decision、scope、actor、reason、risk level、
  transition preview relation、metadata 和 created time。
- 只有 transition preview 为 `ready` 时，才能记录 `approved`；blocked preview 只能记录
  `rejected` 或继续修复上游。

v0.3d 仍不做 promotion apply、不写被管理项目、不创建 execution package、不执行任务。
approval record 是审计事实，不等于 execution。

## Success

- AreaFlow 能创建新 workflow version 候选。
- discussion gate 不通过时不能进入 changes。
- middle-layer、changes、plans、drafts、queue candidate 和 promotion preview 都以 artifact 和 workflow item 表达。
- copy-ready 与 verify-ready artifact 保持分离。
- promotion preview 只生成 live mapping 预览，不写 execution。
- approval record 可审计，但 approval 不等于 execution。
- AreaFlow-owned artifact 写入 local artifact store，不写被管理项目。
- approved approval record 只能在 ready transition preview 后记录，并保持 `approval_is_execution=false`。
