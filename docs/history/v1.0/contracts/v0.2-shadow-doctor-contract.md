# v0.2 Shadow Doctor Contract

## Purpose

本文定义 AreaFlow v0.2 `Shadow Doctor + Drift Check` 的阶段边界。它补充
[`v0.1-import-mirror-contract.md`](./v0.1-import-mirror-contract.md)、
[`areamatrix-import-scope-contract.md`](../../../architecture/areamatrix-import-scope-contract.md)、
[`adapter-profile-boundary.md`](../../../architecture/adapter-profile-boundary.md)、
[`api-surface.md`](./api-surface.md)、
[`security-permissions.md`](./security-permissions.md)、
[`../dogfood/areamatrix-contract.md`](../../../dogfood/areamatrix-contract.md) 和
[`../migration/areamatrix-workflow-migration.md`](../migrations/areamatrix-workflow-migration.md)。

v0.2 的目标不是创建新 workflow version，也不是接管 AreaMatrix `./task-loop`。它只证明 AreaFlow 对
AreaMatrix 当前 workflow metadata 的理解可以被只读复验，并能把 drift、stage coverage、native doctor
授权状态和 readiness bundle 稳定暴露给 CLI、API、Web/Desktop 后续只读界面。

核心规则：v0.2 是 shadow evidence，不是 cutover evidence。任何 doctor、readiness、import diff 或
verification bundle 的 `pass` 都不能被解释为 authoring cutover、execution cutover、project write、
worker execution 或 AI engine 调用已经打开。

## Position In Migration

v0.2 只在 v0.1 之后运行：

```text
Import + Status Mirror
-> Shadow Doctor + Drift Check
-> New Version Authoring
```

进入 v0.2 前必须已经满足：

```text
project add/import 可重复
latest import snapshot 存在
previous import snapshot 可用于 diff，或明确标记 previous_missing
AreaMatrix protected paths 未变化，除非 status projection apply 已单独授权
```

v0.2 完成后只能把系统推进到 v0.3 authoring 设计和 preview；不能直接进入 cutover 或 execution。

## Minimal Closed Loop

v0.2 最小闭环是：

```text
project import
-> project import
-> project doctor
-> project summary
-> project readiness
-> project import-diff
-> project verify-bundle
-> protected path proof
```

`project doctor` 可以在 AreaFlow PostgreSQL 中记录 diagnostic command、event 和 audit metadata。它不得写
AreaMatrix 文件，不得创建 workflow version，不得创建 worker lease，也不得运行 `./task-loop run`。

## Required CLI Commands

v0.2 必须稳定支持：

```text
areaflow project doctor <project_key>
areaflow project doctor <project_key> --json
areaflow project doctor <project_key> --allow-native --json
areaflow project summary <project_key>
areaflow project summary <project_key> --json
areaflow project readiness <project_key>
areaflow project readiness <project_key> --json
areaflow project import-diff <project_key>
areaflow project import-diff <project_key> --json
areaflow project verify-bundle <project_key>
areaflow project verify-bundle <project_key> --json
areaflow project events <project_key>
areaflow project events <project_key> --limit 20
```

`verify-bundle` 是只读验收聚合入口。它读取已有 summary、readiness、import diff 和 events，不应触发新的
doctor run、native command、status projection apply 或 project write。

## API Boundary

v0.2 Query API 最小面：

```text
GET /api/v1/projects/{project_key}/summary
GET /api/v1/projects/{project_key}/readiness
GET /api/v1/projects/{project_key}/import-diff
GET /api/v1/projects/{project_key}/verification-bundle
GET /api/v1/projects/{project_key}/events?limit=20
```

v0.2 diagnostic Command API 或 CLI service action 可以记录 doctor result，但只允许写 AreaFlow-owned
diagnostic state：

```text
command_requests
events
audit_events
doctor summary metadata
```

这些写入是 AreaFlow 自身审计事实，不是被管理项目写入，也不是 workflow runtime execution。

## Doctor Checks

v0.2 `project doctor` 至少覆盖：

```text
adapter:
  project adapter 是否为 areamatrix。

project_root:
  root path 是否存在且可读取。

db_inventory:
  imported workflow_versions、residuals、artifacts、snapshots 和 projection history 是否存在。

import_snapshot:
  latest import snapshot 是否存在。

source_snapshot:
  当前 AreaMatrix source snapshot 是否可从允许路径读取。

import_idempotency:
  同一次 doctor 期间重复读取 adapter snapshot 是否稳定。

hash_drift:
  latest import source hash 是否等于当前 source snapshot hash。

project_config_drift:
  当前 areaflow.yaml 是否等于 AreaFlow 记录的 active config snapshot。

inventory_consistency:
  PostgreSQL inventory 与当前 adapter snapshot 是否一致到 v0.2 要求的 metadata 深度。

stage_coverage:
  AreaMatrix profile stage 是否能映射到当前导入 / source metadata。

native_workflow_doctor:
  AreaMatrix 原生 `./dev workflow doctor` 的授权状态和可选执行结果。
```

`stage_coverage` 只证明 AreaMatrix profile 的 stage 在 metadata 层有覆盖。它不证明每个 stage 的所有
prompt、日志、报告、diff 或 evidence 原文已导入，也不证明 promotion apply 或 execution 可运行。

## Status Semantics

v0.2 状态只允许：

```text
pass
warn
fail
blocked
skipped
unchanged
changed
previous_missing
```

语义约束：

- `warn` 必须保留为 `warn`，不能为了 UI 简化压成 `pass`。
- `skipped` 必须说明原因，例如 native command 未授权。
- `blocked` 必须说明缺少哪个前置条件，例如 status mirror 未授权。
- `unchanged` 只表示本次 import diff 的比较字段未变化，不表示完整 workflow tree 无 drift。
- `pass` 只在对应 check 范围内有效，不向上推导为 cutover 或 execution readiness。

## Native Doctor Boundary

AreaMatrix 原生 doctor 命令固定为：

```text
./dev workflow doctor
```

默认 `project doctor` 不运行 native command，只记录 `native_workflow_doctor=warn/skipped`。只有同时满足以下
条件时才允许运行：

```text
--allow-native 显式传入
project config 允许 run_commands，或 --allow-native 按本次运行覆盖 capability gate
command allowlist 明确允许 ./dev workflow doctor
command 未命中 forbidden_commands
```

`--allow-native` 只能作为一次性人工授权覆盖 `run_commands` capability gate。它不能绕过 command allowlist，
不能绕过 forbidden command，也不能授权任何其他命令。

以下命令在 v0.2 必须保持 blocked / denied：

```text
./task-loop run
git reset --hard
git checkout --
rm -rf
unknown commands not in allowlist
```

运行 native doctor 后仍必须执行 protected path proof，证明 AreaMatrix 受保护路径未被改动。

## Summary Requirements

`project summary --json` 和 API summary 必须至少稳定暴露：

```text
doctor.status
doctor.drift_status
doctor.project_config_drift_status
doctor.stage_coverage_status
doctor.native_doctor_status
import.latest_source_hash
import.previous_source_hash when present
import.history_ready_for_diff
inventory counts
latest event summary
```

UI、脚本和自动化不得通过解析人读 doctor 文本来推导 drift 或 native doctor 状态。

## Readiness Requirements

`project readiness --json` 必须返回：

```text
status
items[]
items[].key
items[].status
items[].message
items[].metadata
```

v0.2 readiness 至少包含：

```text
import_snapshot
import_history
status_mirror
doctor_report
drift_check
project_config_drift
stage_coverage
native_workflow_doctor
events
```

真实 AreaMatrix read-only smoke 中，如果没有授权写 `.areaflow/status.json`，`status_mirror` 可以是
`warn/blocked`。这必须保留为 accepted warning 或 blocker，不能静默转成 pass。

## Import Diff Requirements

`project import-diff --json` 比较 latest 和 previous import snapshot。v0.2 最小比较字段：

```text
source_hash
workflow_version_count
residual_count
artifact_count
task summary
v1 execution summary
status summary hash
project config hash
snapshot created_at ordering
```

v0.2 `import-diff` 不证明：

```text
full workflow tree diff
all prompt bodies diffed
all logs/evidence diffed
native doctor passed
stage semantic equivalence
```

如果 previous snapshot 不存在，状态应为 `previous_missing` 或 readiness blocker，而不是假装 unchanged。

## Verification Bundle

`project verify-bundle --json` / API `verification-bundle` 聚合：

```text
summary
readiness
import_diff
recent_events
phase_gate
accepted_warnings[]
blocked_reasons[]
safety_facts
```

`phase_gate.name` 必须是：

```text
v0.2-shadow-doctor
```

真实 AreaMatrix read-only smoke 允许出现：

```text
native_workflow_doctor warn/skipped because native command was not authorized
status_mirror warn/blocked because real .areaflow/status.json write was not authorized
```

这些可以成为 `accepted_warnings`，但不能从 `phase_gate.blocked_reasons` 中消失，除非对应授权和证据已经存在。

## Allowed Reads

v0.2 允许读取：

```text
AreaFlow PostgreSQL metadata
AreaFlow event / audit metadata
AreaMatrix read envelope defined by areamatrix-import-scope-contract.md
areaflow.yaml current file hash/config
git status for protected path proof
```

v0.2 不读取 AreaMatrix secret，不读取用户文件，不读取或复制历史 artifact 大原文。

## Allowed Writes

v0.2 允许写：

```text
AreaFlow PostgreSQL diagnostic command/event/audit rows
AreaFlow-owned diagnostic artifact metadata, when explicitly produced
```

v0.2 不允许写：

```text
AreaMatrix workflow/README.md
AreaMatrix .areaflow/status.json unless this is a separate authorized v0.1 mirror apply
AreaMatrix workflow/versions/**
AreaMatrix execution/progress/logs/checkpoints/run summaries
AreaMatrix source code
```

## Forbidden Actions

v0.2 必须阻断：

```text
create workflow version
promote queue
write workflow/versions/**
write workflow/versions/*/execution/**
write progress.json
run ./task-loop run
start worker
create worker lease / heartbeat / attempt
call AI engine
resolve secret
write source code or generated project files
run git mutation
publish release or create package
restore apply
```

## Handoff To v0.3

进入 v0.3 前必须证明：

- doctor JSON 字段稳定。
- summary / readiness / import-diff / verify-bundle 可由 CLI 和 API 查询。
- latest 和 previous import snapshot 可解释。
- hash drift、project config drift、stage coverage 和 native doctor status 分层可见。
- native doctor 未授权时只记录 warn/skipped，不越权执行。
- AreaMatrix protected path proof clean，或变化只限单独授权的 status projection apply。
- 没有 workflow/execution/progress/logs/checkpoint/source code 变化。

这些条件只允许打开 v0.3 authoring preview / AreaFlow-authored records。它们不允许打开 AreaMatrix
authoring cutover、execution cutover、worker execution 或 project writes。

## Anti-patterns

v0.2 不能被解释为：

- “doctor warn 已接受，所以 v0.2 gate pass。”
- “import diff unchanged，所以完整 workflow tree 无 drift。”
- “stage coverage pass，所以所有历史 artifact 已迁移。”
- “native doctor skipped，所以 native doctor 等价通过。”
- “`--allow-native` 可以运行任意 AreaMatrix 命令。”
- “verification bundle pass，所以 cutover 可以直接 apply。”
- “readiness 可查询，所以 worker execution 已打开。”
