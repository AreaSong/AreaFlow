# v1.0 Stable Platform Contract

## Purpose

本文定义 AreaFlow v1.0 `Stable Platform` 的阶段合同。它补充
[`completion-audit-contract.md`](../contracts/completion-audit-contract.md)、
[`release-final-gate-contract.md`](../contracts/release-final-gate-contract.md)、
[`artifact-backup-restore-contract.md`](../contracts/artifact-backup-restore-contract.md)、
[`operations-deployment-observability-boundary.md`](../contracts/operations-deployment-observability-boundary.md)、
[`auth-team-secret-boundary.md`](../../../../proposals/auth-team-secret.md)、
[`team-remote-control-boundary.md`](../../../../proposals/team-and-remote-control.md)、
[`plugin-marketplace-boundary.md`](../../../../proposals/plugin-marketplace.md) 和
[`../history/v1.0/milestones/v1.0-stable-platform.md`](../milestones/v1.0-stable-platform.md)。

v1.0 的目标是让 AreaFlow 从可用骨架进入可复验、可诊断、可审计、可发布评审的稳定平台状态。v1.0
不是高风险自动化阶段；它不打开真实 restore apply、release publish、secret resolve、remote worker、
team permission enforcement、unknown plugin execution、remote ops 或 destructive rollback。

核心规则：v1.0 的最终完成证明只能来自 completion audit。release final gate、Web/Desktop 展示、smoke
通过、package preview 或 rollout preview 都不能单独声明 100%。

## Position In Roadmap

v1.0 位于 Desktop Shell 之后，是 0-100% 路线的关闭阶段：

```text
Desktop Shell
-> Stable Platform
-> v1.x high-risk apply ladder
```

v1.0 只允许推进：

```text
backup manifest
artifact integrity
restore dry-run
audit coverage
permission doctor
adapter/profile conformance
plugin seed / marketplace metadata
security boundary readiness
operations readiness
release readiness
release remediation / acceptance / exception preview chain
release final gate
release evidence bundle
release package / distribution / publish / rollout preview
completion audit
AreaMatrix dogfood completion proof
protected path proof
```

v1.0 不允许推进：

```text
restore database apply
project file restore apply
archive copy/upload
object storage upload
retention-aware GC/delete apply
secret plaintext resolve
engine secret injection
API token issuance/enforcement
team permission enforcement
remote worker credential issuance
remote worker dispatch
unknown plugin install/enable/execute
webhook delivery / inbound callback as business fact
external API connector apply
budget quota decrement / usage charge
release package creation
git tag/sign/push
artifact upload
release publish
rollout state write
remote ops control
managed upgrade
destructive rollback
full support bundle export
default remote telemetry
```

## Stable Does Not Mean Apply

v1.0 稳定的是平台边界和证据链，不是所有真实动作：

```text
manifest = inventory, not backup package
restore dry-run = plan, not restore apply
artifact integrity = proof, not repair
archive preview = decision aid, not copy/delete/upload
release final gate = go/no-go, not publish
package preview = manifest/path preview, not archive creation
distribution preview = channel preview, not upload
publish gate = blocker, not release approval
rollout preview = plan, not rollout state
security readiness = gap matrix, not auth enforcement
Desktop/Web display = observation, not operation
completion audit = proof, not apply permission
```

任何需要写数据库、写项目文件、读取 secret、调用外部 API、启动 worker、发布 release、恢复状态或远程控制的
能力，都必须进入 v1.x high-risk apply ladder，并拥有单独 command、approval、rollback/remediation、
audit 和 focused smoke。

## Completion Stack

v1.0 关闭必须同时满足九类证据，具体审计规则以
[`completion-audit-contract.md`](../contracts/completion-audit-contract.md) 为准：

```text
E1 design source alignment
E2 phase and task matrix
E3 command, API, and smoke evidence
E4 AreaMatrix dogfood completion
E5 release and packaging preview
E6 backup, restore, artifact, and retention
E7 operations readiness
E8 security, permission, and isolation
E9 AreaMatrix protected path proof
```

任一类证据缺失、范围不匹配、只由间接证据支撑或仍处于 blocked / missing_evidence，v1.0 不能声明 100%。

## AreaMatrix Dogfood Boundary

AreaMatrix 是 v1.0 的第一 dogfood 项目。100% 不是只看 AreaFlow 自身功能，而是必须证明 AreaMatrix
迁移闭环完成：

```text
Import
-> Mirror
-> Shadow
-> Authoring Cutover
-> Execution Beta
-> Execution Cutover
-> Archive
-> Shim Retirement
```

完成证明必须区分：

```text
readiness / preview:
  只说明下一步需要什么。

cutover apply:
  必须有 explicit approval、command response、event、audit、rollback path 和 protected path proof。

execution forwarding v1:
  只允许 `./task-loop run` 转发 read-only verify、doctor/readiness、artifact evidence、
  status/projection validation 或 release/readiness check 类任务。它证明 AreaFlow 接管执行入口和审计主状态，
  不打开 source write、generated retained write、repair、checkpoint、engine、secret、network、publish 或
  restore apply。

archive gate:
  必须证明历史 workflow / execution metadata 已 immutable indexed，旧 progress / logs / checkpoints 只作为
  reference，历史文件没有删除、移动或重写。

shim retirement:
  只退役旧 task-loop 主执行能力；不能删除 historical workflow / progress / logs / evidence。
```

如果 `execution-cutover-readiness` 仍是 blocked、`execution_cutover_apply_open=false`，或 `./task-loop run`
的 forwarding/retirement 缺少命令映射和 rollback 证据，completion audit 必须返回 incomplete。
如果 forwarding 证据没有声明并验证第一版允许任务范围，或把 read-only / evidence forwarding 解释成自动写
代码、repair、checkpoint、engine、secret、network、publish 或 restore 已打开，completion audit 也必须返回
incomplete 或 blocked。

## Release Boundary

v1.0 release 链路固定为：

```text
release readiness
-> remediation plan
-> acceptance preview
-> acceptance gate
-> exception doctor
-> exception record preview
-> exception schema preview
-> exception migration approval gate
-> exception apply preview
-> final gate
-> evidence bundle
-> package preview
-> distribution preview
-> publish gate
-> publish approval preview
-> rollout plan preview
```

Release final gate `pass` 是 completion audit 的必要条件，但不是充分条件。它只允许进入 evidence /
package / distribution / publish / rollout preview，不允许创建 release package、写 exception record、运行
migration、tag、sign、push、upload 或 publish。

## Backup / Restore / Artifact Boundary

v1.0 必须证明可恢复范围可解释：

```text
PostgreSQL metadata inventory exists
AreaFlow-owned local artifact metadata is covered
local artifact hash/size integrity can be verified
project_reference / external_project remains metadata-only unless accepted exception exists
object artifact remains skipped/needs_attention unless verifier proves it
restore dry-run exposes blockers and guardrails
archive preview does not copy/upload/delete bytes
```

`project_reference`、`external_project` 或 object verifier skipped 不能累计成完整 restore-ready。它们必须进入
`needs_attention`、`blocked` 或显式 release exception 路径。

## Security / Team / Secret / Remote Boundary

v1.0 只允许 schema、readiness、doctor、role matrix、blocked reason 和 audit gap。它必须保持：

```text
auth_enforcement_open=false
team_permission_enforcement_open=false
api_token_issuance_open=false
api_token_enforcement_open=false
secret_resolve_open=false
remote_worker_credentials_open=false
budget_enforcement_open=false
quota_decrement_open=false
webhook_delivery_open=false
inbound_callback_open=false
external_api_call_open=false
authorization_changed=false
secret_plaintext_read=false
```

任何 team/admin/desktop/web/worker UI 的存在都不能改变 API authorization、project write scope、secret scope、
worker credential scope、publish scope 或 restore scope。

## Operations Boundary

v1.0 运维范围是本机可诊断，不是托管运维：

```text
install / migrate / start / project register smoke
health / readiness / doctor / service status
metadata-only support bundle preview
local-only telemetry default
migration ledger readiness
desktop service-control / notification / tray gates remain read-only
```

v1.0 不允许 remote ops control、managed upgrade、destructive rollback、raw support bundle export 或默认远程
telemetry。任何运维命令不得修改 AreaMatrix protected paths。

## Protected Path Boundary

v1.0 completion audit 必须保存 AreaMatrix protected path proof：

```bash
git -C /Users/as/Ai-Project/project/AreaMatrix status --short -- \
  workflow/README.md \
  .areaflow/status.json \
  scripts/task_loop/console.py \
  scripts/dev_tools/cli.py \
  scripts/task_loop/runner.py \
  scripts/areaflow_shim.py \
  workflow/versions \
  workflow/versions/v1-mvp/execution/_shared/progress.json
```

没有明确 cross-repo authorization task 时，该命令必须无输出。任何 historical workflow、progress、logs、
checkpoint、release evidence 或用户文件删除都必须 blocked。

## Go / No-Go

v1.0 可以进入完成审计前，至少需要：

```text
go test ./...
go build ./cmd/areaflow
cd web && npm run build
cd desktop && npm run build
git diff --check -- .
smoke-local or equivalent local fixture evidence
v1-stable fixture evidence
project isolation evidence
AreaMatrix protected path proof
release final gate evidence
completion audit evidence
```

如果需要真实 PostgreSQL、browser、local service 或 AreaMatrix cross-repo proof，证据必须记录 scope、时间、
命令、结果和清理状态。没有运行不能写 pass，只能写 missing_evidence。

## Handoff To v1.x

v1.x 只能在 v1.0 completion audit 完整解释后按 high-risk apply ladder 逐级打开。Deferred 能力必须保留：

```text
status = deferred
boundary contract
risk class
required approval
rollback/remediation path
focused smoke
AreaMatrix first policy
```

v1.0 的任何 preview、gate、readiness、dashboard、desktop panel、smoke 或 release evidence 都不能替代
v1.x task-local apply evidence。

## Anti-patterns

v1.0 必须拒绝这些解释：

- “go test / build 都过了，所以 100%。”
- “release final gate pass，所以 100%。”
- “package preview ready，所以 release package 已创建。”
- “publish gate pass，所以已经可以发布。”
- “restore dry-run ready，所以 restore apply 可以执行。”
- “Desktop 能看到 service status，所以 process control 已打开。”
- “security readiness 存在，所以 team/auth/secret 已启用。”
- “project_reference 有 hash，所以 artifact 完整可恢复。”
- “AreaFlow 自己可运行，所以 AreaMatrix dogfood 完成。”
- “shim readiness pass，所以 task-loop forwarding 或 retirement 完成。”

## Closing Condition

AreaFlow 只有在以下条件全部满足时，才能声明 0-100% 完成：

```text
v1.0 contract evidence complete
release final gate pass
completion audit status complete
AreaMatrix dogfood Import -> Mirror -> Shadow -> Authoring Cutover -> Execution Beta -> Execution Cutover -> Archive -> Shim Retirement proven
no v0-v1.0 task is planned / missing_evidence / blocked
implementation gap audit has no hidden v1.0 blocker
project isolation evidence passes with real PostgreSQL or explicitly recorded required-scope evidence
AreaMatrix protected path proof clean or explicitly authorized in completed cross-repo task
v1.x high-risk capabilities remain closed unless they have their own approved apply packet
```

在此之前，只能声明具体阶段、具体 gate 或具体 preview 的完成状态，不能声明 AreaFlow 0-100% 完成。
