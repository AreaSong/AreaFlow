# v0.7 Web Dashboard Contract

## Purpose

本文定义 AreaFlow v0.7 `Web Dashboard` 的阶段边界。它补充
[`api-surface.md`](./api-surface.md)、
[`v0.6-worker-beta-contract.md`](./v0.6-worker-beta-contract.md)、
[`command-approval-contract.md`](./command-approval-contract.md)、
[`team-remote-control-boundary.md`](../../../../proposals/team-and-remote-control.md) 和
[`../history/v1.0/milestones/v0.7-web-dashboard.md`](../milestones/v0.7-web-dashboard.md)。

v0.7 的目标是把 AreaFlow 的 project、workflow version、stage、run、artifact、residual、approval、
worker、audit 和 release preview 状态通过 React + TypeScript Web dashboard 展示出来。Web 是 API
client 和观察面，不是新的状态源事实，不直接写 PostgreSQL、artifact store、worker state 或被管理项目文件。

核心规则：v0.7 Web 可以展示 Query API、SSE 和只读 gate 的结果；任何写动作必须保持 disabled /
read-only，直到对应 Command API、risk preview、permission preflight、approval、audit outcome 和 focused
smoke 证据稳定并获得单独开闸。

## Position In Roadmap

v0.7 位于 Worker Beta 之后、Multi-project Worker Pool 之前：

```text
Worker Execution Beta
-> Web Dashboard
-> Multi-project Worker Pool
-> Desktop Shell
```

v0.7 只允许推进：

```text
read-only dashboard over /api/v1
project / version / stage / run / artifact / residual panels
approval record display
worker registry display
audit trail display
SSE event observation
run detail visibility guard
Web write action gate display
shim authorization display
shim apply packet/gate display
status projection authorization and Package A packet/gate display
execution cutover readiness display
release preview chain display
```

v0.7 不允许推进：

```text
Web-created approval
Web drain / cancel / start apply
Web artifact archive apply
Web status projection apply
Web generated write apply
AreaMatrix shim editing
execution cutover apply
./task-loop run forwarding
release package / approval / rollout / publish apply
remote Team Console
API token / auth enforcement
secret resolve
worker scheduling apply
direct artifact content reads from local paths
direct PostgreSQL writes
```

Global run and artifact routes must include `project_key` when called from Web. A mismatched `project_key` must return
`404` and must not reveal whether the global ID exists in another project.

Web must not read local artifact store paths, local project roots, `.areaflow/status.json`, `workflow/versions/**`,
`progress.json`, logs or checkpoint files directly. Artifact original content remains behind API-level artifact content
routes and later permission/redaction gates; v0.7 default panels show metadata only.

## Dashboard Panels

v0.7 dashboard panels are read-only:

```text
project picker:
  project list and selected project metadata.

summary:
  project summary, readiness, doctor/drift/stage coverage.

version timeline:
  workflow version labels, lifecycle status and import mode.

stage board:
  workflow items grouped by profile stage.

trace detail:
  selected stage, workflow item, artifact, residual, approval or run context.

run timeline:
  run metadata plus run detail tasks, attempts and artifact metadata.

artifact browser:
  metadata, sha256, size, URI and linked workflow item context.

residual/blocker view:
  residual key, status, source path, impact and close condition.

approval records:
  existing approval records only.

worker status:
  worker registry and heartbeat metadata only.

audit trail:
  action, decision, capability, resource, reason and created_at.

live events:
  SSE / events table observation only.
```

Panels may display v0.6 scoped outcomes such as `passed`, `verified`, `artifact_written` or
`rollback_verified`, but must preserve the scope by showing associated run type, task kind, dry-run flag, summary,
metadata, gate status or safety facts. Web must not reinterpret scoped v0.6 evidence as AreaMatrix execution cutover.

## Write Action Gate

`GET /api/v1/web/write-action-gate` is the v0.7 gate for future Web write controls. It may list actions such as:

```text
approval_record
run_drain
run_cancel
artifact_archive_preview
status_projection_apply
generated_write_apply_beta
```

In v0.7 these actions default to:

```text
default_ui_state=disabled
status=blocked
read_only=true
```

The gate may explain required Command API, command class, risk preview, permission preflight, approval, audit,
affected resources and focused evidence. It must not create command requests, approvals, events, audit events, leases,
attempts or artifacts.

Web buttons for these actions may be visible only as disabled controls or read-only readiness rows. Any enabled write
control requires a later contract update and focused smoke proving that the UI sends a protected Command API request
with idempotency key, request hash, actor, reason, risk, approval/gate result and audit outcome.

## SSE Boundary

SSE is an observation channel only.

Allowed:

```text
append new events to the visible timeline
trigger a read refresh
show connection state
resume with an event cursor
```

Forbidden:

```text
derive canonical run state only from SSE payload
advance workflow lifecycle from browser state
create commands from event handlers
open notification or desktop permissions
store hidden second-state in local storage
```

If SSE disconnects, Web must fall back to Query API refresh or show stale/unknown state. It must not infer success,
failure, approval or completion from missing events.

## Approval Console Boundary

v0.7 may display an approval console shape, but it is read-only:

```text
existing approval records
approval blockers
required approvals
risk and affected resources
audit facts
disabled next actions
```

v0.7 must not create approval records from Web. `approval` remains a Command API write boundary; approval UI opening
requires a later write-action contract and focused evidence that duplicate submit, idempotency conflict, stale
precondition, permission denial and audit failure are handled fail-closed.

## Multi-project And Team Boundary

v0.7 may show multiple local projects and project-scoped status. It is not a remote Team Console.

Allowed:

```text
local project picker
project-scoped query visibility guard
worker pool summary preview
schedule preview display
team / auth / secret readiness as blocked preview if API exposes it
```

Forbidden:

```text
login/session management
API token issuance or revocation
team membership mutation
remote worker credential issuance
secret read / resolve / injection
remote command console
cross-project action without explicit project_key
```

Remote read-only or command console behavior must follow
[`team-remote-control-boundary.md`](../../../../proposals/team-and-remote-control.md) and is outside v0.7 default scope.

## AreaMatrix Boundary

When viewing AreaMatrix, Web may display:

```text
import / mirror / shadow / cutover readiness
shim authorization packet
shim apply packet/gate review
execution cutover readiness
status projection metadata
workflow/run/artifact/audit metadata
```

Web must not:

```text
edit AreaMatrix shim files
write workflow/README.md
write .areaflow/status.json
write workflow/versions/**
modify progress.json, logs or checkpoint
forward ./task-loop run
start legacy runner
open execution cutover apply
```

AreaMatrix write-related panels must surface blockers, allowed files and safety facts from API responses instead of
offering direct file edits.

## Release Preview Boundary

v0.7 may display release final gate, evidence bundle, package preview, distribution preview, publish gate, publish
approval preview and rollout plan preview. These panels are read-only and must preserve preview language.

Web must not create a package, approval, rollout, tag, signature, upload, publish, release exception record or restore
apply. A `pass` in a release gate may unlock the next preview panel, but not a Web write action.

## Handoff To v0.8

v0.7 handoff to v0.8 requires evidence that:

- Web build passes.
- Browser smoke proves dashboard loads through `/api/v1` and SSE.
- Browser smoke fails if Web sends non-GET `/api/v1` requests.
- Project-scoped run detail uses `?project_key=`.
- Worker pool summary and schedule preview are displayed as preview/read-only.
- Web write action gate displays disabled actions and required evidence.
- Shim authorization, shim apply packet/gate, status projection authorization/packet/gate, execution cutover readiness and release preview chain remain read-only.
- Artifact panels show metadata only and do not read local artifact paths.

v0.8 can then expand multi-project worker pool views. v0.8 must not treat v0.7 schedule preview display as real
scheduler apply.

## Smoke Evidence

v0.7 smoke evidence is split by surface:

```text
web build:
  cd web && npm run build

browser smoke:
  scripts/smoke-web.sh
  scripts/smoke-web-check.mjs

real AreaMatrix read-only browser smoke:
  scripts/smoke-web-areamatrix-readonly.sh

write action gate:
  docs/development/web-write-action-gate-evidence.md

cutover / release observation:
  docs/development/execution-cutover-readiness-evidence.md
  docs/development/v1-stable-fixture-evidence.md

compatibility shim observation:
  docs/development/compatibility-shim-readiness-evidence.md
```

Every smoke report must state whether Web issued non-GET `/api/v1` requests or any non-v1 `/api` requests. The real
AreaMatrix read-only browser smoke must additionally show that
`GET /api/v1/projects/areamatrix/status-projections/authorization` returns
`required_authorization_phrase=授权执行 Package A，只允许写 AreaMatrix .areaflow/status.json`, that the UI renders the
same phrase, that release / completion panels render `real_100=blocked`, `claim_scope`, `not_real_100=true`,
`evidence_only=true`, `status_alone_is_not_completion=true`, release-candidate decision text, and the Package A /
execution cutover / archive / shim retirement blockers, and that `.areaflow/status.json`, `workflow/README.md`, protected path git status,
command request/event/audit/gate/status projection/approval/run/task/attempt/artifact/worker/heartbeat/lease counts
remain unchanged during browser observation. Any non-GET request from v0.7 dashboard is a blocker unless the request
belongs to an explicitly opened Web write-action contract; any non-v1 `/api` request is a blocker.

## Anti-patterns

v0.7 must reject these interpretations:

- “Web displays a gate, so the action is approved.”
- “Web displays `rollback_verified`, so real AreaMatrix generated apply is open.”
- “SSE delivered an event, so browser state is canonical.”
- “A visible button can call a service helper directly.”
- “A local artifact URI can be opened by the browser from disk.”
- “Schedule preview means scheduler apply is running.”
- “Release final gate pass means Web can publish.”
- “Dashboard presence means remote Team Console is enabled.”
