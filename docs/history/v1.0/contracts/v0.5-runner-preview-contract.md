# v0.5 Runner Preview Contract

## Purpose

本文定义 AreaFlow v0.5 `Runner Preview` 的阶段边界。它补充
[`v0.4-workflow-ownership-cutover-contract.md`](./v0.4-workflow-ownership-cutover-contract.md)、
[`execution-model.md`](../../../architecture/execution-model.md)、
[`execution-opening-strategy.md`](../../../../proposals/execution-opening.md)、
[`command-approval-contract.md`](./command-approval-contract.md)、
[`api-surface.md`](./api-surface.md) 和
[`../history/v1.0/milestones/v0.5-runner-preview.md`](../milestones/v0.5-runner-preview.md)。

v0.5 的目标是证明 AreaFlow 可以用 PostgreSQL + AreaFlow-owned artifact store 表达 execution
model：run、run_task、run_attempt、preflight、artifact、event、audit event 和 command response。
它是 dry-run 执行预演，不是真实 execution，不替代 `./task-loop run`，不调用 AI engine，不执行 shell，
不领取真实 worker task，不解析 secret，不访问 network，也不写被管理项目文件。

核心规则：v0.5 runner preview 只证明执行模型可表达、可审计、可回放；不能被解释为 worker beta、
task-loop forwarding、AreaMatrix execution cutover、source write、checkpoint、repair 或 publish 已打开。

## Position In Migration

v0.5 位于 authoring cutover 之后、worker execution beta 之前：

```text
Workflow Ownership Cutover
-> Runner Preview
-> Worker Execution Beta
-> Web Dashboard
```

v0.5 只允许推进：

```text
execution model schema -> AreaFlow PostgreSQL
runner preview report -> AreaFlow artifact store
command/audit evidence -> AreaFlow append-only facts
run control -> dry-run DB-only status control
```

v0.5 不允许推进：

```text
worker real execution
project file write
./task-loop run forwarding
AI engine / Codex CLI execution
secret resolution
checkpoint / repair apply
AreaMatrix execution cutover
```

## Required CLI Commands

v0.5 must support:

```text
areaflow run preview <project_key> <version_label> [--json]
areaflow run preview <project_key> <version_label> --risk-level high [--json]
areaflow run start <run_id> [--json]
areaflow run drain <run_id> [--json]
areaflow run cancel <run_id> [--json]
```

`run preview` creates dry-run AreaFlow execution evidence only. `run start`、`run drain` 和 `run cancel`
only control dry-run run status in AreaFlow DB; they must not claim a task, start a worker, run commands,
call an engine or write managed project files.

## API Boundary

v0.5 Query API:

```text
GET /api/v1/runs/{run_id}
GET /api/v1/runs/{run_id}/events
GET /api/v1/artifacts/{artifact_id}
GET /api/v1/artifacts/{artifact_id}/content
```

v0.5 Command API:

```text
POST /api/v1/projects/{project_key}/workflow-versions/{version}/runner-preview
POST /api/v1/runs/{run_id}/start
POST /api/v1/runs/{run_id}/drain
POST /api/v1/runs/{run_id}/cancel
```

These command routes may write AreaFlow PostgreSQL command/event/audit rows, run/task/attempt rows and
AreaFlow-owned artifact store content. They must not write managed project files, start workers, create real worker
leases, run project shell commands, call AI engines, resolve secrets, access the network or forward `./task-loop run`.

## Slices

v0.5 has two slices:

```text
v0.5a:
  runner preview schema, command/API, copy/verify dry-run attempts, runner_preview_report artifact and risk preflight.

v0.5b:
  protected dry-run run control: start, drain and cancel.
```

Both slices remain dry-run. v0.5b is not process control and not worker control; it only changes AreaFlow DB run
status for dry-run runs.

## Runner Preview Shape

`runner.preview` must create or replay a command request using an idempotency key and request hash.

When newly created, it must produce:

```text
run_type = runner_preview
run_kind = execution
run.status = passed
run.dry_run = true
run_task.task_kind = workflow_item_preview
run_task.status = queued
run_task.risk_level = requested risk level
run_attempt attempt_kind = copy
run_attempt attempt_kind = verify
artifact_type = runner_preview_report
event_type = runner.preview.completed
audit action = runner.preview
```

`run.status=passed` means the preview itself completed. It does not mean the queued dry-run task has been executed.
`run_task.status=queued` means v0.6 worker dry-run may later demonstrate lease/claim/release behavior on that task.

## Preflight Requirements

Runner preview preflight must record:

```text
dry_run = pass
workflow_version_authored = pass
write_permissions = pass because writes are not attempted
command_permissions = pass because commands are not executed
secret_permissions = pass because secrets are not resolved
network_permissions = pass because network is not used
risk_gate = pass or blocked
```

High or mission-critical risk with `risk_policy != allow` must be blocked. A blocked preflight must not create
run/task/attempt/artifact rows and must leave an auditable command denial when routed through a protected command path.

## Artifact Boundary

`runner_preview_report` is AreaFlow-owned evidence:

```text
storage_backend = local
artifact_type = runner_preview_report
content_type = application/json
sha256 and size_bytes are recorded in PostgreSQL
URI points to AreaFlow artifact store
```

The API returns artifact metadata and URI, not the report body inline. Artifact content can be read only through the
artifact content route and must remain project-scoped. A run-level artifact may have no `workflow_item_id`; readers must
treat that as run evidence, not as a missing item link.

## Command Response Safety Facts

`runner.preview` response must record:

```text
run_id
run_status
dry_run=true
run_task_ids[]
attempt_ids[]
artifact_ids[]
artifact_type
artifact_sha256
artifact_size_bytes
preflight_status
event_id
audit_event_id
project_write_attempted=false
execution_write_attempted=false
area_matrix_write_attempted=false
engine_call_attempted=false
commands_run=false
secrets_resolved=false
network_used=false
```

`run.start`、`run.drain` 和 `run.cancel` responses must record:

```text
decision
previous_status
status
dry_run=true
task_claimed=false
worker_started=false
project_write_attempted=false
execution_write_attempted=false
engine_call_attempted=false
commands_run=false
secrets_resolved=false
network_used=false
```

Non-dry-run run control must be denied and must also record that no task, worker, command, engine, secret, network or
project write was attempted.

## Allowed Reads

v0.5 may read:

```text
AreaFlow PostgreSQL metadata
AreaFlow project config and permissions
AreaFlow workflow version metadata
AreaFlow artifact store content for runner preview reports
AreaFlow events/audit/command responses
```

v0.5 must not read AreaMatrix source files as part of runner preview. AreaMatrix read-only evidence belongs to later
scoped read-only verify or native doctor boundaries, not to `runner.preview`.

## Allowed Writes

v0.5 may write:

```text
AreaFlow PostgreSQL
AreaFlow command_requests
AreaFlow runs
AreaFlow run_tasks
AreaFlow run_attempts
AreaFlow artifacts metadata
AreaFlow events
AreaFlow audit_events
AreaFlow-owned local artifact store
```

v0.5 must not write:

```text
AreaMatrix files
AreaMatrix workflow/versions/**
AreaMatrix workflow/versions/**/execution/**
AreaMatrix progress.json / logs / checkpoints
AreaMatrix workflow/README.md
AreaMatrix .areaflow/status.json
managed project source code
managed project generated/projection paths
```

## Forbidden Actions

v0.5 must block:

```text
real execution apply
worker run-once real copy / verify
worker lease creation from runner preview
AI engine / Codex CLI execution
shell command execution
secret resolution
network access
managed project reads for runner preview
managed project writes
checkpoint apply
repair apply
restore apply
publish apply
./task-loop run forwarding
AreaMatrix execution cutover
```

## Handoff To v0.6

v0.5 handoff to v0.6 requires evidence that:

- `runner.preview` creates run/task/copy attempt/verify attempt/artifact/event/audit/command response.
- Replaying the same idempotency key returns the existing preview result.
- High-risk preview without `risk_policy=allow` is blocked.
- `runner_preview_report` hash and size match local artifact content.
- `run.start`、`run.drain` and `run.cancel` only control dry-run AreaFlow DB status.
- Non-dry-run run control is denied.
- Safety facts prove no project write, execution write, engine call, shell command, secret, network or AreaMatrix write.
- AreaMatrix protected path proof is clean.

v0.6 can then introduce worker lifecycle and dry-run worker claim/release behavior. It still must not treat v0.5
runner preview as permission to execute real tasks.

## Anti-patterns

v0.5 must reject these interpretations:

- “runner preview passed, so execution is ready.”
- “run.status=passed means the queued task was executed.”
- “dry-run run control can start a worker.”
- “copy and verify attempts exist, so project copy/verify happened.”
- “runner_preview_report in artifact store means AreaMatrix files changed.”
- “runner preview may forward `./task-loop run` because compatibility blocks it.”
- “high-risk preview can be downgraded to warn without explicit `risk_policy=allow`.”
- “v0.5 evidence can be accumulated into real AreaMatrix execution cutover.”
