# v0.2 Shadow Doctor + Drift Check

阶段合同见 [`../architecture/v0.2-shadow-doctor-contract.md`](../architecture/v0.2-shadow-doctor-contract.md)。
本 milestone 只建立 shadow evidence，不打开 authoring cutover、execution cutover、worker execution 或
project write。

## Goal

AreaFlow 能校验自己对 AreaMatrix workflow 的理解是否正确。

## Includes

- Stage coverage report。
- Hash drift check。
- Residual consistency check。
- Import idempotency check。
- AreaMatrix workflow doctor 等价对照。
- Summary、readiness、import-diff、verify-bundle 和 events 的稳定 JSON/API 查询。
- Native doctor 授权边界：默认 skipped/warn，显式 `--allow-native` 仍受 allowlist / forbidden command 限制。

## Initial CLI

```bash
areaflow project doctor areamatrix
areaflow project doctor areamatrix --json
areaflow project doctor areamatrix --allow-native --json
areaflow project summary areamatrix
areaflow project summary areamatrix --json
areaflow project readiness areamatrix
areaflow project readiness areamatrix --json
areaflow project import-diff areamatrix
areaflow project import-diff areamatrix --json
areaflow project verify-bundle areamatrix
areaflow project verify-bundle areamatrix --json
areaflow project events areamatrix
areaflow project events areamatrix --limit 20
GET /api/v1/projects/areamatrix/summary
GET /api/v1/projects/areamatrix/readiness
GET /api/v1/projects/areamatrix/import-diff
GET /api/v1/projects/areamatrix/verification-bundle
GET /api/v1/projects/areamatrix/events?limit=20
```

初始 doctor 只读 PostgreSQL 和 AreaMatrix project root，检查最新 import snapshot、
当前 source snapshot、hash drift、DB inventory consistency、residual consistency、
artifact index consistency、adapter idempotency、import snapshot history 和 AreaMatrix profile stage coverage。

每次 doctor 运行都应写入 AreaFlow `events`，event type 为
`project.doctor.completed`。metadata 保存 overall status、check counts 和每个
check 的结果；这为后续 Web/Desktop dashboard 的 gate timeline 提供数据来源。
`project events` 是初始只读 timeline 入口，用于从同一事件源查看 doctor、import、
status export 等项目事件。
`GET /api/v1/projects/{id}/events` 提供同一数据的 REST/JSON 入口，为后续 Web dashboard
和 desktop shell 复用 timeline 语义。
`project summary` 和 `GET /api/v1/projects/{id}/summary` 聚合 project metadata、
import snapshot、inventory counts 和最新 doctor 状态，作为 Web dashboard 第一屏的
只读状态源。
summary 必须提供稳定字段 `doctor.status`、`doctor.drift_status` 和
`doctor.stage_coverage_status`、`doctor.native_doctor_status`；UI 不需要解析 doctor
metadata 才能显示漂移状态或原生 doctor 对照状态。
`readiness` 汇总 Import、Mirror、Events、Summary API、Doctor、Drift、Stage Coverage
和 Native Doctor 的阶段就绪状态。`doctor --json`、`summary --json` 和
`readiness --json` 是脚本 / agent / 自动化入口，应保持字段稳定，不要让自动化依赖人读文本。
Import 会保留历史 status snapshot，用于后续 snapshot diff；当前索引表仍刷新为最新导入状态。
`import-diff` 初始比较 latest 和 previous import snapshot 的 source hash、version count、
residual count、task summary 和 v1 execution summary；更细的 artifact/residual diff 后续再扩展。
`verify-bundle` 是 v0.2 的只读验收聚合入口，包含 summary、readiness、import diff 和最近
events；它不执行 doctor、不写文件、不改变项目状态。
`verify-bundle.phase_gate` 给出 v0.2 是否可进入下一阶段的只读判定。默认未授权 native
workflow doctor 时，`native_workflow_doctor=warn` 可作为 accepted warning；它不能绕过权限，
也不代表已经执行了原生命令。
真实 AreaMatrix read-only smoke 中，如果 `.areaflow/status.json` mirror apply 未被单独授权，
`status_mirror` 和 `phase_gate` 可以保持 warn/blocked；这必须作为显式 blocker 或 accepted warning
展示，不能被 UI 或脚本压成 pass。

AreaMatrix 原生 `./dev workflow doctor` 是 shadow 对照项。AreaFlow 只有在
`run_commands` capability 和 command allowlist 都允许时才执行；否则
`native_workflow_doctor` 记录为 warn/skipped，不越权运行命令。
`--allow-native` 是一次性的人工授权：它只覆盖本次运行的 `run_commands`
capability 门禁，但仍要求 command allowlist 允许 `./dev workflow doctor`，且不能绕过
forbidden command。

## Excludes

- 创建新 version。
- 写 `workflow/versions/**`。
- 执行任务。
- 把 verification bundle pass 解释为 authoring cutover 或 execution cutover。
- 用 native doctor skipped/warn 伪装 native doctor 等价通过。

## Success

- `areaflow project doctor areamatrix --json` 返回稳定 JSON 字段。
- `areaflow project doctor areamatrix --allow-native --json` 可在显式授权下对照 AreaMatrix 原生 doctor。
- `areaflow project summary areamatrix --json` 提供 `doctor.status`、`doctor.drift_status`、`doctor.stage_coverage_status` 和 `doctor.native_doctor_status`。
- summary import 字段提供 `history_ready_for_diff` 和 previous source hash 信息。
- `areaflow project import-diff areamatrix --json` 能输出 latest/previous import snapshot 的稳定差异报告。
- `areaflow project verify-bundle areamatrix --json` 能一次性输出 v0.2 验收所需的 summary、readiness、import diff 和 events。
- `verify-bundle.phase_gate.status` 为 `pass` 或 `blocked`；默认 native doctor skipped/warn 只能进入 `accepted_warnings`，不能成为静默执行。
- `areaflow project events areamatrix` 能从同一事件源查看 import、status export 和 doctor。
- REST API 提供 summary、events 和 readiness 的只读入口。
- readiness 提供稳定字段 `status` 和 `items[]`，每个 item 包含 `key`、`status`、`message` 和 `metadata`。
- 未授权 native doctor 时只记录 skipped/warn，不越权执行命令。
- `verify-bundle.phase_gate` 保留真实 `pass` / `blocked` 状态；warn/skipped 不被静默升级。
- protected path proof 证明 AreaMatrix `workflow/README.md`、`.areaflow/status.json`、`workflow/versions/**`
  和 v1 `progress.json` 未被 v0.2 doctor 改动，除非另有 status projection apply 授权证据。
